## 3. Implementation Methodology

The repeatable implementation model must include discovery, data/evidence readiness, sandbox/demo, tenant/customer setup planning, governed UAT, support onboarding, go-live decision, hypercare, BAU handoff, and customer success tracking.

Every implementation must preserve the AIM evidence-first workflow and cannot bypass AI staging, engineer review, deterministic calculation governance, report issue gates, work-order gates, audit logging, object-storage evidence controls, or human approval requirements.

## 4. Human Authority Boundary

AI/n8n/service actors cannot approve delivery role assignments.
AI/n8n/service actors cannot approve implementation methodology.
AI/n8n/service actors cannot approve scale operating model readiness.
AI/n8n/service actors cannot sign scale operating model closure.

AIM remains the system of record. n8n remains orchestration-only.
